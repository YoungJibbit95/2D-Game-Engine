# Engine Status

This is a compact inventory of the current engine state. Open work remains in the checklist files; this file is allowed to mention completed systems for orientation.

Current validated snapshot (2026-07-15): post-integration Debug and Release builds/full suites pass with 0 warnings/errors and 995/995 tests; format, performance, strict asset/preview, isolated publish and seven client-smoke gates pass locally. `LoadedGameSession` owns one `GameSimulation`, named RNG/event persistence and data-driven living-world/runtime-animation content; `PlayingState` submits commands and renders prepared snapshots/commands. Hosted CI remains unverified.

## July 14 Large Update

- Mouse-first menus now use release-inside clicks, pointer capture and drag sliders, with keyboard/gamepad fallback, accessibility controls, rounded geometry, gradients, glow, shadows and scene-capture backdrop blur.
- Presentation now includes tile-aware 2D ray-cast sunlight/point-light shadows, colored torches, AO, penumbra, bloom, cave residual light, bounded screen-space water reflections, biome parallax and ambient/weather particles. This is not hardware raytracing.
- Surface/cave lighting follows deterministic local terrain height instead of global spawn Y. Open, Unknown and Occluded sky states prevent unloaded chunks from leaking light; materialization/unload re-dirties dependents, visible regions are scheduled first, open sky remains bright and shafts/cave mouths receive bounded lateral skylight falloff.
- Combat adds authoritative guard stamina, directional block, parry, guard break, combined projectile/contact resolution, attack runtime contracts and projectiles with homing, gravity, drag, pierce and bounce.
- The fixed-tick animation foundation supports clips, events, blends, action locks, layered rigs and attachments. The central content database validates eight Wave 04 clips, a five-layer player rig, action mappings, 14 entity profiles and typed fallbacks; active player and entity preparation consume this registry, and a 200-entity preparation loop is 0 B steady state.
- Spawning uses activity-source rings around the current play area rather than world spawn. Habitat, loaded-chunk, viewport, liquid, ground, collision and cap checks drive deterministic friendly/hostile populations across negative and positive X.
- Infinite generation materializes data-driven structure templates across chunk boundaries. Amber Grove and Twilight Marsh plus Amber Workshop and Marsh Lantern Grove extend the regional living-world sample.
- `LivingWorldRuntime` owns the phased world-event executor and bounded journal. `world-events.json` atomically persists active state, cooldowns, journal and the exact-once player-action sequence with backup/legacy recovery; deterministic action triggers plus rare/quantity loot modifiers are active across all authoritative loot paths. Audio routing is active but no production sound files ship.
- Authoritative gameplay outcomes now feed fixed-capacity visual/audio queues. Normal and rare loot drops, pickup, mining/building, combat/projectile hits, deaths, crafting, resource/status changes and scheduled/action world events expose presentation commands and queue telemetry without growing per-frame collections.
- Replay diagnostics capture versioned fixed-step input frames, optional item-use requests and periodic state hashes in a bounded ring; JSON round trips are size-limited and first-divergence reports identify the failing tick/reason plus the last valid checkpoint.
- Background streaming now classifies failures, retries transient jobs with bounded exponential backoff and enforces operation, elapsed-time and decoded-byte main-thread apply budgets. All limits are validated persistent settings with Pause-menu sliders.
- The reusable combat runtime now advances startup, active, recovery and cooldown ticks with input buffering, combo/cancel windows, multiple timed swept shapes, resource metadata and bounded typed events. Selected-item weapon execution is the next integration boundary.
- Production Wave 04 contributes 31 contract-complete assets. Wave 05 adds 24 assets/122 explicit frames, tiles 12-15, 10 items, 7 recipes, four actors with loot/spawn, two structures and three events. The static website separates engine/game/download information from a searchable 25-article wiki.
- The current representative quick tick is 0.099 ms average, 0.131 ms p95, 0.346 ms p99 and 7,250 B average allocation; calibration records 0.065 ms average and 6,237 B. Both remain far below the 4 ms/16,384 B regression gates; frame snapshots and entity queries remain the primary allocation debt.

## What Exists

The project has a solid MonoGame/.NET 8 shell with a clean `Game.Core` separation. Core already covers chunked world storage, finite and horizontally infinite world bounds, budgeted chunk streaming, chunk metadata, tile coordinate helpers, deterministic world generation, streamed chunk save/load, packed chunk region files, coordinated save/autosave/load orchestration, project/session bootstrapping, persistent equipment/status effects/character appearance, content registries, sprite asset manifests and audits, runtime sprite animation clips, data-driven item actions with typed feedback results/events, crafting, farming, topdown maps, dialogue, shops, loot, combat, projectiles, entities, spawning, settings, engine-neutral UI animation/layout primitives, underground-aware lighting, time, spatial queries, batched tile mutation results, a reusable performance profiler, and a core runtime simulation tick with world-simulation scheduling.

The engine now has a first explicit game-project boundary. `yjse.game.json` describes a concrete game, `Game.Core.Projects` resolves content/mod/save paths, and `Game.Core.Sessions.GameSessionBootstrapper` creates or resumes playable sessions from project root, save directory, seed, world name, settings, startup, and content data. Every loaded session owns one simulation and one shared runtime object graph. The MonoGame client can run an external game repository via `YJSE_GAME_ROOT` or a loose content root via `YJSE_GAME_DATA`, while local `Game.Data` remains sample/dev content. Startup profiles own starter inventory, selected hotbar slot, startup map, and default world profile.

The client now has YjsE branding, a main menu, world-select and create-world screens, a loading state that can resume existing saves, a keyboard/mouse-driven settings menu, an in-game pause menu, a playable state, camera follow, debug overlays, hotbar UI, a Terraria-like inventory overlay with visible slots, cursor stack, quick move, live equipment slots, stat summary, and item tooltips, a crafting overlay with recipe category tabs, recipe list, ingredient/station checks, craft button, and shift-repeat crafting, visible tile/liquid rendering, and mouse-based selected item use for mining, building, melee, ranged, magic casting, and mana consumables. Escape no longer closes the game from the main menu by accident. Resolution, fullscreen, and VSync settings apply live through the client shell. Rendering is no longer only rectangle-driven: player, enemy, dropped item, projectile, inventory, hotbar, crafting result, and ingredient draw paths now resolve real sprite assets where available and fall back safely. There is a first pass at render layers, shader registry scaffolding, post-processing settings, a texture registry that can resolve sprite assets to PNGs or deterministic placeholders, a day/night and depth-aware parallax background renderer for forest, night forest, magical grove, cave, and deep cave scenes, and a chunk render command cache with visible/rebuilt/evicted draw metrics plus a settings-driven cache budget. Terrain tile commands now carry autotile masks into sprite-frame selection, and dirt/grass/stone/wood/leaves use 16-mask blockface source frames plus matching AI sprite briefs for 256x16 terrain strips. The character/ambient-life asset wave now includes player action frames, v2 body/hair/clothing/accessory sheets, rabbit, bird, bat, cave worm, foliage props, furniture/decor/tool/UI/icon assets, corrected 512x128 parallax backgrounds, and first magic item/projectile sprites.

World generation is modular and deterministic. It now generates surface terrain, dirt/stone layers, cave walkers, large organic caverns with connector tunnels, profile-driven ore veins, shaped surface lakes, cave pools, underground walls with cave-core cleanup, regional structures, pass-through mineable trees and a safe spawn point. The sample regional set covers Forest, Meadow, Amber Grove, Twilight Marsh, Mushroom Cave, Crystal Depths and Deep Cave. `WorldGenerationProfile` exposes validated tuning for each pass, while `WorldAnalyzer` reports cavern regions, liquid bodies, surface/cave liquid and wall coverage. `InfiniteWorldChunkGenerator` streams deterministic chunks in negative and positive X while keeping vertical bounds finite and dimension-banded. `ChunkStreamingService` builds immutable visibility requests, limits concurrent load/save jobs, rejects cancelled or stale session results, applies completed work under a strict update budget, saves dirty snapshots before unload, and exposes queue, byte, operation, failure and timing telemetry.

The engine also has increasingly useful debug foundations: `EngineDebugSnapshotBuilder`, `/debug world`, `WorldAnalyzer`, dirty-region tracking, global event observation, a bounded `GameEventJournal`, an engine-neutral `PerformanceProfiler` with rolling averages/peaks/budgets/allocation samples, save result events, world raycasts, line-of-sight checks, rectangle/circle/cone entity queries, and tile flood fill. The client exposes live pass timings, allocation rows, event history, streaming backlogs, and existing world/render/save metrics through optional overlays.

YjsE now has explicit non-Terraria genre modules. `Game.Core.Farming` supports Stardew-like loops with crop JSON definitions, seed-to-crop lookup, tilled/watered plot state, planting, water-gated daily growth, seasonal restrictions, harvesting, regrow crops, inventory integration, explicit failure statuses for UI feedback, farm plot save/load, and selected-item farming action routing. `TopDownMovementController` adds gravity-free movement over the same tile collision model. `Game.Core.Maps` adds fixed topdown map definitions with layers, collision, objects, interactables, spawn points, warps, registry validation, content/mod loading, map sessions, map-specific movement, runtime object state, interaction targeting, object action resolution, and transition services for RPG/life-sim maps. `Game.Core.Dialogue` and `Game.Core.Shops` provide the first NPC/social/economy content layer with graph validation, session advancement, shop stock, sell prices, and inventory-backed buy/sell transactions.

## Partial Systems

Rendering still needs texture atlases, render-target-backed chunk batching, compiled HLSL effect integration, actual draw-call/texture-switch/GPU counters and calibrated 1080p/1440p traces. The active presentation path already has bounded render targets, tile-aware light masks, colored point lights, reflections, blur, particles, multi-layer parallax, source-rectangle character animation and entity visual command preparation.

World simulation has liquid data, baseline liquid flow, dirty-region scheduling, event-driven tile mutation tracking, integration into `GameSimulation`, and chunk metadata refreshes. Dirty-region clamping now preserves negative X in horizontally infinite worlds, and liquid seeding scans loaded chunks instead of pretending an infinite world has a finite horizontal span. It still needs pressure/settling behavior, richer liquid rules, particle scheduling, renderer-cache integration, and better visual water surfaces.

Interactions can target and use selected hotbar items, including pass-through mineable trees and non-solid placeables such as workbenches and `mangrove_root`. Item use returns explicit succeeded/in-progress/blocked states, failure reasons, cooldown telemetry, mining progress, restores and applied effects. Mining publishes deduplicated lifecycle events; mined light occluders have Core/client stale-shadow regressions. `BuildingSystem` uses the transactional placement service for validated, atomic world/inventory commits, including authoritative inventory and negative X. Remaining work is durable request/reconciliation IDs, richer anchors, sideview tile entities/furniture, walls and liquid rules.

Farming is now wired into the live client path for hoe, watering can, seed use, placeholder plot/crop drawing, daily growth, and autosave/load. It still needs proper sprite crop rendering, farm action UI feedback, weather/rain watering, shops, shipping bins, NPC schedules, and client/tooling support for authored topdown farm/town maps.

Combat can damage enemies with melee/projectiles, resolve deterministic loot and defend the player through guard, parry, mitigation and guard break. Magic consumes/scales mana, status effects are data-driven, and advanced projectiles expose homing, gravity, drag, pierce and bounce. The reusable attack sequencer is fully tested, but the active item-use path still needs to consume authored sequences and materialize attacks only inside active windows; line-of-sight must be universal and guard/projectile lifecycle feedback still needs complete presentation routing.

World generation still needs smooth biome transition bands, broader biome-specific wall/foliage materials, structure spacing/rarity rules, chest rooms and loot placement, seed retry integration, preview tooling and sampled infinite-world quality gates. Structure templates are now data-driven rows/legends with explicit air and surface/cave origins and materialize across chunk and negative-X boundaries.

UI has mouse-first click/drag semantics, pointer capture, focus states, keyboard/gamepad navigation, sliders, dropdowns, segmented controls, toggles, scrolling and delayed tooltips. Settings cover video, rendering, UI effects, audio, gameplay, keybinds, accessibility and debug. Inventory/crafting remain Terraria-like dense surfaces with transaction-backed drag/split/merge/favorites/search/batch craft. It still needs full automated click-path smokes, persisted recipe knowledge/pins, font asset/shaping work and migration of more overlays onto reusable core widget models.
