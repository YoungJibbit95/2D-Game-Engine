# Engine Status

This is a compact inventory of the current engine state. Open work remains in the checklist files; this file is allowed to mention completed systems for orientation.

Current validated snapshot (2026-07-22): Debug and Release solution builds pass with 0 warnings/errors, and functional coverage passes 1,710/1,710 in both configurations. Twenty-one performance classes containing 39 tests pass in fresh Release processes at unchanged budgets; combined-host p99 outliers remain diagnostic. Day Forest, readable blue night, Frostwood snow and 3440x1440 ultrawide smokes pass without permanent darkness, opaque weather bands or lower-image parallax repetition. `LoadedGameSession` still owns one authoritative `GameSimulation`; hosted CI remains unverified.

## What Exists

The project has a solid MonoGame/.NET 8 shell with a clean `Game.Core` separation. Core covers chunked/infinite worlds, streaming and persistence, deterministic generation, content/asset contracts, actions, crafting, farming/topdown/dialogue/shop modules, combat/projectiles/entities/spawning, renderer-neutral lighting and particle physics, equipment mobility, settings, UI models, spatial queries and the authoritative fixed simulation host.

The engine now has a first explicit game-project boundary. `yjse.game.json` describes a concrete game, `Game.Core.Projects` resolves content/mod/save paths, and `Game.Core.Sessions.GameSessionBootstrapper` creates or resumes playable sessions from project root, save directory, seed, world name, settings, startup, and content data. Every loaded session owns one simulation and one shared runtime object graph. The MonoGame client can run an external game repository via `YJSE_GAME_ROOT` or a loose content root via `YJSE_GAME_DATA`, while local `Game.Data` remains sample/dev content. Startup profiles own starter inventory, selected hotbar slot, startup map, and default world profile.

The client has YjsE branding, mouse/gamepad menus, world creation/loading, pause/settings, camera follow, developer overlays, hotbar, Terraria-style inventory/crafting, equipment/stat/mana UI and selected-item mining/building/melee/ranged/magic use. The camera now glides through a critically damped deadzone; the HUD exposes Double Jump/Flight/Glide from one shared sheet. V7 feature planes plus V6 depth stacks cover the viewport without stretching or lower-edge repetition, while prepared terrain/liquid/lighting/particle commands keep simulation authority out of Draw.

World generation is modular and deterministic. Topology V2 gives finite and infinite worlds the same broad, slope-limited surface and coherent dirt-depth sampling, while the finite pipeline also generates cave walkers, large organic caverns with connector tunnels, profile-driven ore veins, shaped surface lakes, cave pools and underground walls with cave-core cleanup. Both paths support regional structures, pass-through mineable trees and a safe spawn point. The sample regional set covers Forest, Meadow, Amber Grove, Twilight Marsh, Mushroom Cave, Crystal Depths and Deep Cave. `WorldGenerationProfile` exposes validated tuning for each pass, while `WorldAnalyzer` reports cavern regions, liquid bodies, surface/cave liquid and wall coverage. `InfiniteWorldChunkGenerator` streams deterministic chunks in negative and positive X while keeping vertical bounds finite and dimension-banded. `ChunkStreamingService` builds immutable visibility requests, limits concurrent load/save jobs, rejects cancelled or stale session results, applies completed work under a strict update budget, saves dirty snapshots before unload, and exposes queue, byte, operation, failure and timing telemetry.

The engine also has increasingly useful debug foundations: `EngineDebugSnapshotBuilder`, `/debug world`, `WorldAnalyzer`, dirty-region tracking, global event observation, a bounded `GameEventJournal`, an engine-neutral `PerformanceProfiler` with rolling averages/peaks/budgets/allocation samples, save result events, world raycasts, line-of-sight checks, rectangle/circle/cone entity queries, and tile flood fill. The client exposes live pass timings, allocation rows, event history, streaming backlogs, and existing world/render/save metrics through optional overlays.

YjsE now has explicit non-Terraria genre modules. `Game.Core.Farming` supports Stardew-like loops with crop JSON definitions, seed-to-crop lookup, tilled/watered plot state, planting, water-gated daily growth, seasonal restrictions, harvesting, regrow crops, inventory integration, explicit failure statuses for UI feedback, farm plot save/load, and selected-item farming action routing. `TopDownMovementController` adds gravity-free movement over the same tile collision model. `Game.Core.Maps` adds fixed topdown map definitions with layers, collision, objects, interactables, spawn points, warps, registry validation, content/mod loading, map sessions, map-specific movement, runtime object state, interaction targeting, object action resolution, and transition services for RPG/life-sim maps. `Game.Core.Dialogue` and `Game.Core.Shops` provide the first NPC/social/economy content layer with graph validation, session advancement, shop stock, sell prices, and inventory-backed buy/sell transactions.

## Partial Systems

Rendering has a compiled pass/resource graph, tile-frame atlas pages, stable material submission, prepared Kawase blur, a physical transient-target pool, direct/diffuse 2D solar transport, temporal shadow reprojection, dynamic light collection, wet-surface radiance and five-plane authored backgrounds. It still needs global graph-level pool ownership, cross-category atlases, material normal/transmission maps, production compiled effects, backend GPU timestamp queries and a complete 1440p biome/quality matrix.

World simulation has liquid data, active-cell flow, bounded compatibility-region seeding, dirty-region scheduling, reusable workspaces, event-driven tile mutation tracking, integration into `GameSimulation`, and chunk metadata refreshes. Dirty-region clamping preserves negative X in horizontally infinite worlds, and liquid seeding scans loaded chunks instead of pretending an infinite world has a finite horizontal span. Prepared biome body/depth/surface/shore presentation is active outside Draw. Simulation still needs pressure/settling behavior, richer liquid rules and particle scheduling; presentation still needs a visible water-heavy acceptance matrix.

Physics has renderer-neutral body types, mass/force/impulse integration, damping/materials/layers, swept full/one-way/half/slope tile collision, allocation-free broadphase and bounded body TOI. EntityManager owns exact contact slices, mass/resistance-aware knockback and fail-closed recovery. `Game.Core.Particles` adds a separate fixed-capacity deterministic pool with generation-safe handles, gravity/wind/drag, swept circle-tile collision, bounce/friction, work budgets, snapshots and bounded events; liquid forces, joints and dense authoritative combined fixtures remain open.

Interactions target selected hotbar actions, foreground tiles and exposed background walls. Mining keeps hardness/tool requirements but applies `GlobalDurationFactor = 0.25`, making valid actions four times faster; completed wall mining clears `WallId` and returns typed drops. Transactional building keeps reach/support/collision and atomic world/inventory commits. Remaining work is durable reconciliation IDs, richer furniture anchors and expanded wall/liquid placement.

Farming is now wired into the live client path for hoe, watering can, seed use, placeholder plot/crop drawing, daily growth, and autosave/load. It still needs proper sprite crop rendering, farm action UI feedback, weather/rain watering, shops, shipping bins, NPC schedules, and client/tooling support for authored topdown farm/town maps.

Combat damages enemies and the player through melee, swept projectiles, contact, guard/parry and typed status effects. Magic spends/scales mana; wands now produce authoritative damage requests and exact wall/entity sweeps. Homing, gravity, drag, pierce and bounce are active, with owner/friendly-fire filters and realistic bounded knockback. Session 0 found four unresolved authority/content contracts: the active combined damage path does not consistently receive equipment/status mitigation, bows still bind one fixed ammo ID instead of an authored ammo policy, axes have no distinct chop action/tool-target contract, and attack-shape JSON lacks finite/positive/bounds validation. These remain `partial`; complete terminal feedback, balance distributions and production audio are also open.

World generation still needs shared chunk-cell cave networks and depression-planned surface lakes in the Infinite path, smooth biome elevation transition bands, broader biome-specific wall/foliage materials, structure spacing/rarity rules, chest rooms and loot placement, seed retry integration, preview tooling and sampled infinite-world quality gates. Structure templates are data-driven rows/legends with explicit air and surface/cave origins and materialize across chunk and negative-X boundaries.

UI has mouse-first click/drag semantics, pointer capture, focus states, keyboard/gamepad navigation, sliders, dropdowns, segmented controls, toggles, scrolling and delayed tooltips. Settings cover video, rendering, UI effects, audio, gameplay, keybinds, accessibility and debug. Inventory/crafting remain Terraria-like dense surfaces with transaction-backed drag/split/merge/favorites/search/batch craft. `Video.UiScale` is currently persisted but not consumed by a shared viewport/inverse-pointer transform, and the 320x180 Main Menu overflows. Deterministic Inventory/Crafting smoke hooks, 720p/1080p/1440p captures, persisted recipe knowledge/pins and font/shaping work remain open.

The sample content audit currently counts 49 items, 32 recipes, 13 loot tables, 6 projectiles, 3 effects, one shop with two offers and six starter items. Only 21 items are directly reachable through the audited startup/recipe/loot/shop sources (43 if ore mining is assumed). `double_jump_boots`, `ether_glider` and `skyward_wings` have active runtime art/contracts but no declared acquisition path; poison arrows are crafted/dropped but cannot satisfy the current fixed bow-ammo contract. These are progression gaps, not verified gameplay loops.
